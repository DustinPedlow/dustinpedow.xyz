A sshlockout Tutorial

Oct 29, 2023
2 min read
Security Tutorials

This is a appreciation article to sshlockout. Yep I’m a shill.

The sshlockout program is a extra layer of security that bans IP addresses that have reached a maximum threshold of failed password attempts. The sshlockout program originates from DragonFlyBSD and has been ported to OpenBSD. If you’re from the Linux world this is similar to Fail2Ban and is a very simple program. When sshlockout is setup using the settings shown in the manpage it will ban any IP address that exceeds 5 incorrect passwords in one hour. Then the IP’s block is expired after 1 day and it is allow to login again.

Now with that all being said let’s get started with the installation!


Installation

After installing sshlockout onto OpenBSD I followed the synopsis within the manpage closely.

I added

table <lockout> persist
block in quick on egress proto tcp from <lockout> to port 2222

into /etc/pf.conf


I then reloaded pf with the new config using

pfctl -f /etc/pf.conf

I added

auth.info;authpriv.info | exec /usr/bin/doas -n /usr/local/sbin/sshlockout -pf "lockout"

into /etc/syslog.conf


The last part of the synopsis was about adding a line into the root crontab to expire blocked IP addresses. I didn’t do this since I want them to be blocked forever but if you want to the line is as follows

3 3 * * *       pfctl -tlockout -T expire 86400

Testing

The way I tested if sshlockout is working is by trying to login from a different IP address. I VPN’d into Mullvad and I moved my ssh keys into a different directory so the ssh command would not use them. I then tried to ssh into my server and for credentials I typed in random passwords with the username root. After 5 failed attempts when I try to use ssh it will just hang instead of asking for my credentials. To escape this it required user intervention with a Ctrl+C.


After disconnecting from my VPN I ssh’d into my remote webserver and ran the command

pfctl -t lockout -T show | grep 221.146.62.21

Sure enough it returned the VPN’s IP address which confirmed to me that it had been added to the blocklist.
